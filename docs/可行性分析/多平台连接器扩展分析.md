## 一、现有架构分析

### 1.1 连接器系统架构

史丽慧小助理 项目已经具备了完善的**可扩展连接器架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                     Gateway (网关)                          │
│              消息路由 / 会话管理 / Tab 管理                    │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│               ConnectorManager (连接器管理器)                 │
│      注册 / 启停 / 健康检查 / 消息入站出站转发 / 配对管理        │
└─────────────────────┬───────────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
┌───────────┐ ┌───────────┐ ┌───────────┐
│  Feishu   │ │  DingTalk │ │  WeChat   │ ...
│ Connector │ │ Connector │ │ Connector │
└───────────┘ └───────────┘ └───────────┘
```

### 1.2 已定义的连接器类型

在 `src/types/connector.ts` 中已预定义：

```typescript
export type ConnectorId = 'feishu' | 'dingtalk' | 'wechat' | 'slack';
```

### 1.3 Connector 接口规范

| 接口模块 | 方法 | 说明 |
|---------|------|------|
| 基本信息 | id, name, version | 连接器标识 |

## 二、各平台可行性分析

### 2.1 钉钉 (DingTalk)

| 维度 | 评估 |
|------|------|
| **技术可行性** | ✅ **高** - 官方支持 WebSocket 长连接 |
| **API 完整度** | ✅ 完整 - 支持单聊、群聊、消息收发、图片文件 |
| **SDK 支持** | ✅ 官方 Java SDK，社区有 Node.js 实现 |
| **实现复杂度** | ⭐⭐ 中等 |

**技术方案**：

```typescript
// 钉钉连接器核心实现思路
class DingTalkConnector implements Connector {
  readonly id = 'dingtalk' as const;
  readonly name = '钉钉';
  
  private streamClient: DingTalkStreamClient; // WebSocket 客户端
  
  async start(): Promise<void> {
    // 使用 Stream 模式建立 WebSocket 长连接
    // 参考: https://open.dingtalk.com/document/development/introduction-to-stream-mode
    this.streamClient = new DingTalkStreamClient({
      clientId: this.config.clientId,
      clientSecret: this.config.clientSecret,
    });
    
    // 订阅消息事件
    this.streamClient.on('chat.message', (data) => {
      this.handleIncomingMessage(data);
    });
    
    await this.streamClient.start();
  }
  
  // 发送消息使用 REST API
  outbound.sendMessage = async (params) => {
    // POST https://api.dingtalk.com/v1.0/robot/oToMessages/batchSend
    // 或群消息: /v1.0/robot/groupMessages/send
  };
}
```

**关键 API**：

| 功能 | API 端点 | 文档链接 |
|------|---------|----------|
| Stream 模式 | WebSocket 长连接 | [Stream模式文档](https://open.dingtalk.com/document/development/introduction-to-stream-mode) |
| 单聊消息 | `/v1.0/robot/oToMessages/batchSend` | [单聊文档](https://open.dingtalk.com/document/dingstart/the-application-robot-in-the-enterprise-sends-a-single-chat) |
| 群聊消息 | `/v1.0/robot/groupMessages/send` | [群聊文档](https://open.dingtalk.com/document/orgapp/the-use-of-internal-application-robots-in-person-to-person-single-chat) |
| 上传媒体 | `/v1.0/robot/mediaFiles/upload` | - |

**所需配置**：

```typescript
interface DingTalkConnectorConfig extends ConnectorConfig {
  clientId: string;      // 应用的 ClientId
  clientSecret: string;  // 应用的 ClientSecret
  robotCode?: string;    // 机器人码（可选）
}
```

### 2.2 企业微信 (WeCom)

| 维度 | 评估 |
|------|------|
| **技术可行性** | ⚠️ **中等** - 智能机器人功能有限制 |
| **API 完整度** | ⚠️ 部分 - 需要区分不同类型 |
| **SDK 支持** | ✅ 官方 SDK |
| **实现复杂度** | ⭐⭐⭐ 较高 |

**企业微信机器人类型对比**：

| 类型 | 接收消息 | 发送消息 | 适用场景 |
|------|---------|---------|---------|
| **群机器人** | ❌ 仅 Webhook 被动 | ✅ Webhook 主动 | 单向通知 |
| **智能机器人** | ✅ 长连接回调 | ✅ API 发送 | 双向对话 |
| **企业内部应用** | ✅ 回调配置 | ✅ API 发送 | 完整功能 |

**推荐方案：智能机器人（长连接模式）**

```typescript
class WeComConnector implements Connector {
  readonly id = 'wechat' as const;
  readonly name = '企业微信';
  
  async start(): Promise<void> {
    // 使用智能机器人长连接
    // 参考: https://developer.work.weixin.qq.com/document/path/101463
    // 企业微信通过 WebSocket 推送消息回调
    this.wsClient = new WeComWSClient({
      corpId: this.config.corpId,
      agentId: this.config.agentId,
      secret: this.config.secret,
    });
    
    // 监听消息回调
    this.wsClient.on('aibot_msg_callback', (data) => {
      this.handleIncomingMessage(data);
    });
  }
}
```

**关键 API**：

| 功能 | API 端点 | 文档链接 |
|------|---------|----------|
| 智能机器人长连接 | WebSocket | [智能机器人长连接](https://developer.work.weixin.qq.com/document/path/101463) |
| 发送应用消息 | `/cgi-bin/message/send` | [消息发送](https://developer.work.weixin.qq.com/document/path/90236) |
| 群机器人 Webhook | POST webhook_url | [消息推送配置](https://developer.work.weixin.qq.com/document/path/99110) |

**所需配置**：

```typescript
interface WeComConnectorConfig extends ConnectorConfig {
  corpId: string;        // 企业 ID
  agentId: string;       // 应用 AgentId
  secret: string;        // 应用 Secret
  token?: string;        // 回调 Token（可选）
  encodingAESKey?: string; // 加密密钥（可选）
}
```

**限制说明**：

- 智能机器人目前**仅支持内部应用**
- 群聊消息需要**@机器人**才能触发
- 每个用户每天主动消息有限额

### 2.3 QQ 机器人

| 维度 | 评估 |
|------|------|
| **技术可行性** | ✅ **高** - 官方开放平台支持 |
| **API 完整度** | ✅ 完整 - 支持群聊、频道、私聊 |
| **SDK 支持** | ✅ 官方 Go/Python/Node.js SDK |
| **实现复杂度** | ⭐⭐ 中等 |

**QQ 机器人场景支持**：

| 场景 | 企业主体 | 个人主体 | 说明 |
|------|---------|---------|------|
| 单聊对话 | ✅ | ✅ | 私信功能 |
| 群聊场景 | ✅ | ✅ | QQ 群 |
| 频道场景 | ✅ | ✅ | QQ 频道 |

**技术方案**：

```typescript
class QQConnector implements Connector {
  readonly id = 'qq' as const;
  readonly name = 'QQ机器人';
  
  async start(): Promise<void> {
    // QQ 机器人支持 WebSocket 和 Webhook 两种模式
    // 参考: https://bot.q.qq.com/wiki/develop/api-v2/
    
    // 方案1: WebSocket 模式（推荐）
    this.wsClient = new QQBotWSClient({
      appId: this.config.appId,
      appSecret: this.config.appSecret,
      intents: [
        'GROUP_AT_MESSAGE_CREATE',  // 群聊@消息
        'DIRECT_MESSAGE_CREATE',     // 私聊消息
        'C2C_MESSAGE_CREATE',        // 单聊消息
      ],
    });
    
    this.wsClient.on('ready', () => console.log('QQ Bot Ready'));
    this.wsClient.on('message', (data) => this.handleIncomingMessage(data));
    
    await this.wsClient.connect();
  }
  
  outbound.sendMessage = async (params) => {
    // 群消息: POST /api/v2/channels/{channel_id}/messages
    // 私聊: POST /api/v2/users/{user_id}/messages
    // 需要先获取 access_token
  };
}
```

**关键 API**：

| 功能 | API 端点 | 文档链接 |
|------|---------|----------|
| 获取 Access Token | `/api/v2/app/getAppAccessToken` | [认证文档](https://bot.q.qq.com/wiki/develop/api-v2/) |
| 发送群消息 | `/api/v2/channels/{channel_id}/messages` | [群组模块](https://bot.q.qq.com/wiki/develop/api-v2/server-inter/group/) |
| 发送私聊 | `/api/v2/users/{user_id}/messages` | [私信文档](https://bot.q.qq.com/wiki/develop/gosdk/api/direct_msg/post_direct_message.html) |
| 上传图片 | `/api/v2/channels/{channel_id}/files` | - |

**官方 SDK**：

| 语言 | 仓库地址 |
|------|---------|
| Go | [botgo](https://github.com/botgo/botgo) |
| Python | [botpy](https://github.com/botpy/botpy) |

### 2.4 微信个人号

| 维度 | 评估 |
|------|------|
| **技术可行性** | ⚠️ **风险高** - 无官方 API，依赖第三方方案 |
| **API 完整度** | ❌ 无官方支持 |
| **SDK 支持** | ⚠️ 社区方案（存在风险） |
| **实现复杂度** | ⭐⭐⭐⭐ 高风险 |

**⚠️ 重要提示**：微信个人号机器人**没有官方开放 API**，现有方案存在以下风险：

| 风险类型 | 说明 |
|---------|------|
| **封号风险** | 使用非官方接口可能导致账号被封 |
| **协议不稳定** | 微信协议随时可能更新，导致失效 |
| **法律风险** | 可能违反微信用户协议 |

**社区方案对比**：

| 方案 | 技术原理 | 风险等级 | 链接 |
|------|---------|---------|------|
| **Wechaty** | iPad 协议/Android协议 | ⚠️ 中高 | [github.com/wechaty/wechaty](https://github.com/wechaty/wechaty) |
| **ComWechat** | PC 微信 Hook | ⚠️ 高 | 非开源 |
| **gewechat** | Android 协议 | ⚠️ 中高 | 付费服务 |

**如果需要支持微信个人号，建议**：

```typescript
// 仅作为技术参考，不推荐生产环境使用
class WeChatPersonalConnector implements Connector {
  readonly id = 'wechat_personal' as const;
  readonly name = '微信个人号';
  
  async start(): Promise<void> {
    // 使用 Wechaty（需要 puppet 服务）
    const { WechatyBuilder } = require('wechaty');
    
    this.bot = WechatyBuilder.build({
      puppet: 'wechaty-puppet-service', // 需要付费 puppet 服务
    });
    
    this.bot.on('message', (msg) => {
      this.handleIncomingMessage(msg);
    });
    
    await this.bot.start();
  }
}
```

**建议**：对于企业场景，**强烈推荐使用企业微信**而非个人微信。

### 2.5 Slack

| 维度 | 评估 |
|------|------|
| **技术可行性** | ✅ **高** - 官方 Socket Mode 支持 |
| **API 完整度** | ✅ 完整 - 成熟的 Bot API |
| **SDK 支持** | ✅ 官方 Node.js SDK (@slack/web-api) |
| **实现复杂度** | ⭐⭐ 中等 |

**技术方案**：

```typescript
import { App } from '@slack/bolt';

class SlackConnector implements Connector {
  readonly id = 'slack' as const;
  readonly name = 'Slack';
  
  private app: App;
  
  async start(): Promise<void> {
    // 使用 Slack Bolt 框架（Socket Mode）
    // 参考: https://docs.slack.dev/apis/events-api/using-socket-mode/
    this.app = new App({
      token: this.config.botToken,
      appToken: this.config.appToken,
      socketMode: true, // 启用 Socket Mode，无需公网 IP
    });
    
    // 监听消息事件
    this.app.message(async ({ message, say }) => {
      await this.handleIncomingMessage(message);
    });
    
    // 监听 @机器人 事件
    this.app.event('app_mention', async ({ event, say }) => {
      await this.handleIncomingMessage(event);
    });
    
    await this.app.start();
  }
  
  outbound.sendMessage = async (params) => {
    await this.app.client.chat.postMessage({
      channel: params.conversationId,
      text: params.content,
    });
  };
}
```

**所需配置**：

```typescript
interface SlackConnectorConfig extends ConnectorConfig {
  botToken: string;     // xoxb-xxx
  appToken: string;     // xapp-xxx (Socket Mode 需要)
  signingSecret: string; // 签名验证
}
```

## 七、总结

史丽慧小助理 的连接器架构设计**已经具备了良好的扩展性**，实现多平台支持主要工作是：

1. **实现 Connector 接口** - 每个平台约 300-500 行代码
2. **处理消息格式转换** - 各平台消息格式适配
3. **配置存储扩展** - 前端配置页面支持
4. **前端 UI 适配** - 设置页面添加各平台配置入口

**推荐优先实现顺序**：钉钉 > Slack > 企业微信 > QQ机器人

微信个人号因无官方 API 支持，风险较高，仅建议作为**可选功能**，且需要在 UI 上明确提示用户风险。
